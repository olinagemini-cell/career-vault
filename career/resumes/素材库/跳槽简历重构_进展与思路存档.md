## 一、目标定位

**核心身份：AI赋能的超级PM**

走"AI + 创新"路线，继续深耕项目管理领域，以AI能力放大PM价值，形成差异化竞争力。

- 目标岗位方向：AI + 数字化转型负责人
- AI能力定位：战略层 — 能从0到1规划AI赋能方案并推动落地
- 投递范围：外企/跨国公司、咨询公司/IT服务商、AI/科技创业公司、国内大厂

## 二、现有简历诊断（4个结构性问题）

| 问题 | 具体表现 | 影响 |
|------|---------|------|
| **定位模糊** | 优势亮点堆砌6个维度（战略、资源、预算、合规、AI、方案设计），没有主线 | HR看完不知道你"到底是谁"，无法快速归类 |
| **AI能力被淹没** | AI实践被埋在工作经历的bullet point里，和常规职责混在一起 | 最大差异化优势反而成了"附加信息" |
| **内容严重重复** | BPCE上线、ROI第一、AI应用在"亮点""工作经历""项目经验"三处几乎原文重复 | 4页简历，信息密度只有2页 |
| **叙事无层次** | 每段经历都是"职责+成就"平铺，14年职业路径看不出成长弧线 | 读不出"为什么这个人的下一步应该是AI+数字化转型负责人" |

## 四、管理能力短板的重新定义

### 现状

没有长时间管理过直属下属，没有传统意义上的"带团队"经验。

### 解决策略：不回避，而是重新定义

用**"影响力领导力"（Leading Without Authority）**替代传统"团队管理"叙事。

核心论点：在矩阵式组织和跨境环境里，影响力领导比行政管理更有价值。

### 支撑场景

- BPCE：在成本分摊模式下，跟欧洲总部谈判争取全球资源分配
- BPCE：担任中国业务与欧洲IT架构师之间的战略桥梁，协调50+利益相关方
- NCR：同时管理15+并行项目，协调跨职能团队（销售、运营、开发）
- NCR：在RFP/QBR会议中担任专家角色，跨职能通道建设
- 百胜项目：协调销售、交付、财务三方完成账户转型

### 简历措辞方向

- 不写"管理X人团队"
- 写"跨组织影响力领导""跨境资源谈判""矩阵式团队激励""无授权推动落地"

## 五、多版本简历策略

### 整体架构：1个基础版 + 3个定向版

所有版本共享核心素材，各自调整焦距和篇幅分配。

### 共享模块（所有版本复用）

- 14年PM经验
- $50M+投资组合管理
- 跨境交付能力
- PMP / ITIL / 精益六西格玛 / 大模型应用工程师
- BPCE全球首个上线
- NCR连续3年ROI第一
- 零合规违规
- 影响力领导力（跨组织、跨层级、无授权推动）

### 版本A：AI战略 + 转型负责人

- **目标岗位**：咨询公司AI实践负责人、企业数字化转型总监
- **核心突出**：从0到1规划AI赋能方案、方法论建设、团队赋能
- **AI专区占最大篇幅**，PM经验作为"底盘"
- **叙事重心**：我不只是用了AI，我建立了框架、推动了团队采纳、能复制到其他团队和项目

### 版本B：AI落地 + 高级PM

- **目标岗位**：大厂PMO、外企高级PM / 项目群经理
- **核心突出**：扎实的PM执行力 + AI作为"加速器"
- **PM能力是主角**，AI是差异化加分项
- **叙事重心**：$50M投资组合、全球首个上线、连续3年ROI，比其他PM多了AI这项能力
- **市场需求量最大**，最容易投

### 版本C：数字化转型产品经理

- **目标岗位**：AI创业公司产品负责人、企业数字化转型产品经理
- **核心突出**：业务流程再造、系统设计、用户采纳
- **把"项目经理"叙事转化为"产品思维"叙事**
- **叙事重心**：BPCE系统本地化 = 产品化过程，EPM 30%效率提升 = 产品驱动的业务价值

## 六、新简历结构设计（通用基础版）

### 目标：从4页精简到2页，信息密度翻倍

#### 第一部分：个人信息 + 一句话定位（3行）

```
刘小妍 | AI赋能的战略型项目群经理
14年端到端IT项目群领导力 | 跨境数字化转型 | AI驱动的项目管理创新
```

#### 第二部分：核心优势（4条，取代现在的12条）

1. **AI赋能项目管理先行者** — 团队首位系统性应用AI工具的PM，从0到1建立AI辅助项目管理框架（覆盖需求分析、风险评估、文档生成），效率提升20-25%，框架被团队采纳为标准方法论
2. **跨境数字化转型实战** — 独立领导法国BPCE集团全球首个系统上线项目（3个核心业务系统），中国部署被采纳为全球可复制模板，推动70%运营成本削减路线图
3. **$50M+投资组合管理** — 管理大中华区15+并行项目投资组合，连续3年ROI排名亚太第一，通过跨客户需求整合和资源优化实现
4. **端到端战略交付能力** — 从商业论证到部署后优化全流程独立负责，6年零合规违规，PMP/ITIL/精益六西格玛/大模型应用工程师四证持有

#### 第三部分：AI能力专区（新增，单独成块）

```
【方法论层】
建立可复用的AI集成项目管理框架：工具选型矩阵 × 场景映射 × 效果评估方法论
→ 被团队采纳为标准参考框架

【应用层 — 4个核心场景】
① 需求分析加速：Perplexity供应商竞品分析和行业基准研究 → 周期缩短15%
② 项目全生命周期管理：飞书多维表格搭建数据管理中枢 → 日常管理时间节省25%
③ 风险评估与决策支持：Claude/Gemini风险场景模拟和交付计划优化 → 决策质量提升
④ 文档与沟通提效：AI辅助利益相关方沟通文档、复盘报告生成 → 文档效率提升20%+

【认证层】
大模型应用开发工程师（IITC，2025-2026在读）
```

#### 第四部分：工作经历（精简重组）

- **BPCE（2022.11-至今）**— 占50%篇幅，"跨境数字化转型 + AI创新"双线叙事
- **NCR（2016.09-2022.11）**— 占30%篇幅，聚焦投资组合管理 + EPM效率提升 + ROI
- **药明康德 + 赫兹** — 各2-3行，只保留支撑能力的关键点

#### 第五部分：项目经验

- 取消独立的"项目经验"板块，关键成果整合到对应公司的工作经历中
- 如目标公司要求，可另备项目清单附件

#### 第六部分：教育 + 认证 + 语言（精简为3行）

## 七、关键措辞优化方向

### 需要强化的表达

| 现在的写法 | 建议改为 |
|-----------|---------|
| "团队内首位系统性应用AI工具的项目经理" | "从0到1建立AI辅助项目管理工作流，覆盖需求分析→风险评估→文档生成→复盘分析全链路" |
| "建立可复用的AI集成项目管理工作流框架" | "建立AI+PM方法论框架（含工具矩阵、场景映射、ROI评估），被团队采纳为后续项目标准参考" |
| "持续应用Claude和Gemini进行项目风险评估" | 具体场景化："用Claude进行交付风险场景模拟，提前识别3个关键路径风险并制定缓解方案" |
| "战略型项目群经理" | "AI赋能的战略型项目群经理" |

### 需要砍掉的内容

- 重复出现3次的BPCE成就描述 → 只在最合适的位置出现1次
- "零合规违规"多处重复 → 只在核心优势中提一次
- 赫兹的具体职责细节 → 精简为2行
- "附加信息"中的"MS Office, VISIO, MSP" → 删除
- 社会活动（抗疫志愿者）→ 与目标定位无关，删除
