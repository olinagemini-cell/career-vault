## 🚀 使用流程

### Step 1: 分析JD需求
```
1. 识别JD中的高频关键词
2. 判断职位类型（数字化转型/PMO/Commercial IT/云迁移等）
3. 确定需要哪些技能模块
```

### Step 2: 选择对应模块
```
必选：基础信息模块 + 教育认证模块
核心：根据职位类型选择1-2个核心模块
补充：根据JD关键词选择1-2个补充模块
```

### Step 3: 组合简历内容
```
1. 从基础信息模块提取个人定位
2. 从核心模块提取主要工作经历
3. 从补充模块提取相关经验
4. 确保模块间内容一致性
```

### Step 4: 针对性优化
```
1. 根据JD关键词调整权重
2. 量化成果优先排列
3. 调整案例描述以匹配目标公司
4. 保持诚实，不夸大经验
```

## 📝 模块使用原则

### ✅ 必须遵守的原则
1. **诚实第一**: 不夸大经验，不虚构经历
2. **概念迁移**: 可以强调可迁移的能力，但不能说"有经验"
3. **量化成果**: 优先使用具体数据
4. **价值导向**: 每个模块都说明对雇主的价值

### ⚠️ 需要小心的表述
| 可以说 | 需要避免 |
|-------|---------|
| "熟悉CRM核心业务流程" | "有Salesforce实施经验" |
| "了解云迁移方法论" | "有云迁移项目经验" |
| "具备可迁移的能力" | "精通某某技术" |
| "正在学习AWS" | "AWS认证专家" |
