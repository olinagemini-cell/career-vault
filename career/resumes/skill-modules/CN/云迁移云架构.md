## 技能定义

云迁移是将应用程序、数据和业务流程从本地数据中心迁移到云平台（公有云、私有云或混合云）的战略性技术项目，涉及评估、规划、迁移和优化四个阶段。

## 云迁移核心知识框架

### 主流云平台
| 平台 | 市场份额 | 学习优先级 |
|------|---------|-----------|
| AWS (Amazon Web Services) | 最高 | ★★★★★ |
| Microsoft Azure | 高 | ★★★★☆ |
| Google Cloud Platform (GCP) | 中 | ★★★☆☆ |
| 阿里云 | 国内市场 | ★★★☆☆ |

### 核心服务组件
- **计算**: EC2/AWS, VMs/Azure, GCE/GCP
- **存储**: S3/AWS, Blob/Azure, Cloud Storage/GCP
- **数据库**: RDS/AWS, SQL Azure, Cloud SQL/GCP
- **网络**: VPC, Load Balancer, CDN
- **安全**: IAM, KMS, Security Hub

### 云迁移策略
| 策略 | 说明 | 适用场景 |
|------|------|---------|
| Rehost（直接迁移） | 应用直接搬到云 | 快速迁移，低风险 |
| Replatform（平台迁移） | 轻量优化后迁移 | 部分优化，中等风险 |
| Refactor（重构） | 为云重新设计 | 充分利用云特性，高风险 |
| Retire（淘汰） | 不迁移，直接下线 | 无用应用 |
| Retain（保留） | 暂不迁移 | 关键应用或合规要求 |

### 知识框架笔记

**云迁移核心概念**（面试必备）
```
【云服务模型】
- IaaS（基础设施即服务）：AWS EC2, Azure VMs
- PaaS（平台即服务）：AWS Elastic Beanstalk, Azure App Service
- SaaS（软件即服务）：Salesforce, Office 365

【云部署模型】
- 公有云：AWS, Azure, GCP
- 私有云：企业自建云环境
- 混合云：公有云 + 私有云 + 本地数据中心
- 多云：使用多个公有云服务商

【6R迁移策略】
1. Rehost（直接迁移）：Lift and Shift
2. Replatform（平台迁移）：轻量优化后迁移
3. Refactor（重构）：为云重新设计
4. Repurchase（重新购买）：改用SaaS
5. Retire（淘汰）：直接下线
6. Retain（保留）：暂不迁移

【云迁移4阶段】
1. 评估（Assessment）：应用评估、TCO分析
2. 规划（Planning）：迁移路线图、风险规划
3. 迁移（Migration）：数据迁移、应用迁移
4. 优化（Optimization）：成本优化、性能优化
```

**Q: 如何评估云迁移的可行性？**
A: 方法论回答：
1. 应用评估：技术复杂性、依赖关系、迁移成本
2. 业务价值：敏捷性、成本、性能
3. 风险评估：合规、安全、业务连续性
4. ROI分析：迁移成本 vs 运营成本节约
