> **创建时间**: 2026-05-06
> **核心价值**: 以低代码平台（飞书多维表格）为中枢，集成AI工具（Claude Code）驱动项目全生命周期数据治理；具备Power BI运营dashboard需求分析能力，以ROI与业务使用率数据驱动需求优先级论证
> **来源**: 强生医疗 JD 分析会话沉淀

## 技能定义

**数据驱动治理与低代码平台**是指使用低代码/无代码工具构建项目数据管理中枢，结合AI工具实现工作流自动化，并通过BI工具（Power BI）需求分析对接企业级报表体系，最终以数据驱动需求优先级论证和项目决策的能力。

### 核心能力要素
- 低代码平台（飞书多维表格）搭建项目全生命周期数据管理中枢
- AI工具（Claude Code）驱动低代码平台，实现智能工作流
- Power BI / BI工具的需求分析能力（dashboard需求、HQ模板对齐）
- ROI + 业务使用率数据驱动的需求优先级论证
- 跨系统数据打通（飞书 ↔ Cloud ↔ BI工具）

### 与相邻模块的区别
- **vs 业务分析BA**：BA侧重需求访谈→文档→验证；本模块侧重**数据中枢搭建 + 数据驱动决策**
- **vs AI赋能项目管理**：AI赋能侧重知识库沉淀+效率提升；本模块侧重**低代码平台 + AI驱动工作流**
- **vs 投资组合管理**：投资组合侧重多项目资源协调；本模块侧重**单项目/跨项目的数据治理基础设施**

## 现有经验映射

### BPCE Equipment Solutions | 2022/11-至今

#### 1. 低代码项目数据管理中枢（飞书多维表格）

**搭建内容**：
- 以飞书多维表格为基座，构建项目全生命周期数据管理中枢
- 覆盖维度：需求清单、测试用例、风险登记、变更记录、里程碑跟踪
- 替代此前分散在Excel/邮件中的信息孤岛

**数据驱动实践**：
- 以ROI数据 + 业务使用率数据驱动需求优先级论证
- 实时追踪各需求的实际业务使用情况，用数据支撑"哪些功能该先做"
- 将主观的"我觉得这个重要"转化为客观的"数据说这个覆盖X%业务"

**量化成果**：
- 日常管理时间节省约25%
- 需求优先级决策从"开会争论"变为"数据说话"

#### 2. Power BI 运营Dashboard需求分析

**实际经验**（注意诚实边界）：
- 主导Power BI运营dashboard的**需求分析**与HQ模板对齐
- 角色：需求方（BA），不是开发方（Developer）
- 工作内容：
  - 与HQ BI团队对齐dashboard模板标准
  - 梳理中国区运营指标需求（哪些KPI需要可视化）
  - 定义数据源、刷新频率、权限控制需求
  - 验收dashboard交付物，确保满足业务需求

**诚实边界**：
- ✅ 可以说"主导Power BI运营dashboard需求分析与HQ模板对齐"
- ✅ 可以说"熟练使用数据化工具进行数据建模与可视化"
- ❌ 不能说"精通Power BI开发/DAX/数据建模"
- ❌ 不能说"独立开发Power BI dashboard"
- 🔶 面试时诚实说："我负责需求侧——定义KPI、对齐模板、验收交付；开发由BI团队完成。但我理解Power BI的数据模型和dashboard设计逻辑，能精准翻译业务需求。"

#### 3. Claude Code 驱动飞书多维表格

**具体能力**：
- 使用Claude Code操作飞书多维表格API，实现自动化数据读写
- 场景举例：
  - 自动从飞书表格提取风险/issue数据，生成周报
  - 批量更新需求状态，驱动飞书表格的字段联动
  - AI辅助分析表格中的数据趋势（如缺陷率、需求落地率）

**技术路径**：
- 飞书开放平台API → Claude Code MCP工具链 → 飞书多维表格读写
- 这不是传统开发（没写Python脚本），而是AI驱动的无代码集成

**诚实边界**：
- ✅ 可以说"集成AI工具至低代码平台，实现智能工作流"
- ✅ 可以说"搭建飞书与Cloud的数据通道"
- ❌ 不能说"开发了飞书API集成系统"（她用的是Claude Code MCP，不是自己写代码）
- 🔶 面试时说："我用Claude Code的MCP工具链打通了飞书多维表格，实现了项目数据的AI驱动管理——不需要写代码，但需要理解数据模型和API逻辑。"

## 知识框架

### 工具矩阵

| 工具 | 角色定位 | 熟练度 |
|------|---------|--------|
| 飞书多维表格 | 数据管理中枢（搭建者） | 精通 |
| Power BI | 需求分析方（非开发者） | 需求层面熟练 |
| Claude Code | AI驱动引擎（MCP工具链） | 熟练 |
| Excel | 辅助分析工具 | 精通 |

### 方法论：数据驱动需求优先级论证

```
1. 定义维度：ROI影响、业务使用率、合规必要性、技术依赖
2. 数据采集：飞书多维表格追踪各需求的实际业务使用数据
3. 评分矩阵：四维度加权评分
4. 可视化输出：飞书dashboard / Power BI报表呈现
5. 决策对齐：用数据与业务sponsor对齐优先级，不是"我觉得"
```

## 面试准备

### 常见问题

**Q: 你的Power BI经验是开发还是需求？**
A: "我是需求侧——定义运营KPI体系、对齐HQ dashboard模板、验收BI交付物。我理解Power BI的数据模型逻辑，能精准把业务需求翻译为BI开发规格。如果岗位需要更深的Power BI开发能力，我有快速学习的基础。"

**Q: 你说用Claude Code驱动飞书多维表格，具体是怎么做的？**
A: "飞书开放平台提供了多维表格的API，Claude Code有MCP工具链可以调用这些API。我设计了数据模型（表格结构、字段关系、权限），然后通过Claude Code实现自动化的数据读写和分析——不需要写代码，但需要理解数据模型和API逻辑。这让我能快速搭建项目数据中枢，而不依赖开发团队排期。"

**Q: 低代码平台的经验能迁移到其他工具吗？**
A: "能。飞书多维表格的核心逻辑——数据模型设计、字段关联、视图配置、权限控制——和其他低代码平台（如Airtable、Monday.com、Smartsheet）是相通的。我理解的是数据治理的思维，不是特定工具的操作。"
