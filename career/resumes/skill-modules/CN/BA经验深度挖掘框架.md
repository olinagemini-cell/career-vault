# BA经验深度挖掘框架

> **用途**: 系统回忆和梳理你的BA实践经验
> **目标**: 建立个人BA方法论和完整案例库
> **创建时间**: 2026-04-18

---

## 挖掘策略：按BA生命周期逐环节回忆

### Phase 1: 需求发现与调研（你是如何发现问题的？）

#### 引导问题
**在BPCE，当你接到一个新项目时，你通常会：**
```
1. 第一步做什么？（谁告诉你要做这个项目？）
2. 你会找谁聊？（列出具体角色：销售总监？运营经理？财务？）
3. 你会问什么问题？（能否回忆1-2个具体的访谈场景？）
4. 你如何记录？（笔记？录音？会后整理？）
5. 你如何确认自己理解对了？（复述确认？发邮件总结？）
```

#### 具体案例回忆：销售报价系统
```
【场景回忆】
- 谁发起的？销售总监抱怨什么？
- 你和谁聊过？（销售总监、运营经理、财务经理...）
- 他们分别说了什么？
  * 销售总监关注什么？（效率？合规？）
  * 销售代表抱怨什么？（手工操作慢？容易出错？）
  * 财务担心什么？（合规？审计？）
- 你如何把这些零散的抱怨转化为结构化的需求？
```

### Phase 2: 需求分析与建模（你如何整理和分析需求？）

#### 引导问题
**收集完信息后，你通常会：**
```
1. 如何整理信息？（Excel？Visio？飞书？）
2. 是否画过流程图？（能否回忆一个具体的流程图？）
3. 如何识别关键问题？（有没有用过因果分析？鱼骨图？）
4. 如何区分轻重缓急？（优先级是如何确定的？）
```

#### BPCE具体场景
```
【端到端流程梳理】
销售咨询 → 报价生成 → 审批流程 → 合同签订 → 数据同步

你是否画过类似的流程图？
- AS-IS流程（现状）是什么样的？
- TO-BE流程（目标）是什么样的？
- 你如何说服业务方接受你的流程设计？
```

### Phase 3: 需求文档化（你如何把需求写下来？）

#### 你写过的文档类型
根据你的回答，你写过：
- ✅ 流程图/原型
- ✅ 用户故事/Backlog
- ✅ 测试用例/UAT

#### 深度挖掘问题
```
【流程图/原型】
1. 你用什么工具画流程图？（Visio？ProcessOn？飞书？）
2. 能否回忆一个具体的流程图？（几步？涉及几个角色？）
3. 业务方看懂了吗？有没有需要修改的地方？

【用户故事/Backlog】
1. 你是在什么场景下写用户故事的？（敏捷项目？）
2. 能否回忆一个用户故事的例子？
   格式：作为<角色>，我想要<功能>，以便<价值>
3. 你如何排列Backlog优先级？

【测试用例/UAT】
1. 你如何设计UAT测试用例？
2. 能否回忆一个具体的测试场景？
3. 业务方如何验收？签字还是邮件确认？
```

---

### Phase 4: 需求验证与确认（你如何确保需求是对的？）

#### 引导问题
```
1. 你如何让业务方确认需求？（开会？邮件？原型演示？）
2. 业务方有没有改变主意过？你怎么处理的？
3. 开发过程中，需求有没有变化？你如何控制？
4. 上线后，你如何验证需求被满足了？
```

#### BPCE案例回忆
```
【需求确认场景】
- 你是否开过需求评审会？
- 业务方是否签字确认了？
- 开发过程中，需求有没有变更？
  * 如果有，变更是什么？
  * 你如何评估变更影响？
  * 你如何和业务方/开发团队沟通？
```

---

### Phase 5: 需求追溯与闭环（从需求到成果的完整链路）

#### 引导问题
```
1. 你如何追踪需求的实现状态？（Excel？Jira？飞书？）
2. 上线后，你如何验证需求确实被满足了？
3. 有没有需求上线后发现不对的案例？如何处理？
4. BPCE的2个月内100%采纳率，你具体做了什么？
```

## 按公司深度挖掘

#### 背景
- 你是中国区唯一的PM/BA
- 对接销售、运营、财务全平台
- 主导3个核心系统：报价工具、业务发起系统、数据库系统

#### 深度挖掘问题
```
【报价工具项目】
1. 你还记得第一次和销售总监聊报价系统的场景吗？

   * 你问了什么问题？

2. 需求调研花了多久？

   * 开了几次会？
   * 有没有去现场观察销售工作？

3. 你如何整理这些需求？
   * 是否用过Excel整理需求列表？
   * 是否画过流程图？
   * 是否做过优先级排序？

4. 需求文档是什么样的？
   * 是Word文档还是PPT？
   * 包含哪些部分？
   * 业务方看懂了吗？

5. 开发过程中，需求变了吗？
   * 有没有具体的变更案例？
   * 你如何处理的？

6. UAT你是怎么做的？
   * 谁来测试？测了多久？
   * 发现了什么问题？
   * 如何确认可以上线？

【业务发起系统项目】
1. 这个项目的需求调研是怎么做的？
2. 和报价工具有什么不同？

【数据库系统项目】
1. 主数据管理，你的需求是什么？
2. 如何确保数据一致性？
```

## 快速回忆工具

### 回忆触发词
```
看这些词，能否触发你的回忆：

【需求相关】
- 访谈、沟通、需求
- 流程图、业务流程
- 文档、BRD、FRD、规格说明
- 评审、确认、签字
- 变更、修改、调整
- UAT、测试、验收

【场景相关】
- 销售报价、审批流程
- 定价规则、折扣
- 系统集成、数据同步
- 成本优化、效率提升
- 用户培训、采纳率

【工具相关】
- Visio、流程图工具
- Excel、需求列表
- 飞书、协作文档
- PPT、需求演示
```

### 回忆方法
```
1. **时间线回忆**：
   按时间顺序，回忆每个项目的关键节点

2. **角色回忆**：
   想想你对接过的人，他们分别提过什么需求

3. **冲突回忆**：
   回忆有争议、有困难的需求场景

4. **成功回忆**：
   回忆最有成就感的需求案例

5. **文档回忆**：
   你还保留的文档（邮件、PPT、Excel）
```

## 输出成果

### 我们要产出的内容
```
1. 3-5个完整的BA案例（STAR格式）
   - 每个案例包含：背景、任务、行动、结果

2. BA技能清单
   - 你会的技能 + 具体应用场景

3. BA文档示例
   - 你写过的文档类型 + 具体内容回忆

4. BA方法论
   - 你是如何做BA的（个人实践总结）
```
