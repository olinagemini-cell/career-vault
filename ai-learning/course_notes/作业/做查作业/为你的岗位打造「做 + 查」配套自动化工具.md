# 为你的岗位打造「做 + 查」配套 AI 工具

> 完成时间：建议 7 天内完成，每天 60–90 分钟
> 工具要求：Claude Code（已完成安装）
> 核心方法论：拆解一个真实工作场景 → 做「执行步骤」Skills 技能包 + 做「质量检查」Commands 斜杠命令 → 测试跑通

---

## ⚠️ 先搞清楚：Skills 和 Commands 是两种不同的东西

很多人会混淆这两个概念，先看这张表，**看懂了再往下走**：

| 对比维度     | Skills（技能包）                           | Commands（斜杠命令）                     |
| -------- | ------------------------------------- | ---------------------------------- |
| **是什么**  | 一套完整的技能包                              | 一个简单的斜杠命令                          |
| **目录位置** | `C:\Users\你的用户名\.claude\skills\文件夹名\` | `C:\Users\你的用户名\.claude\commands\` |
| **文件结构** | 要创建文件夹 + 里面放 `SKILL.md` 文件            | 只要创建一个 `.md` 文件                    |
| **文件命名** | 文件【夹】名就是技能名（如 `grade`）                | 文件【名】就是命令名（如 `check-dashboard.md`） |
| **调用方式** | 输入 `/文件夹名` 或用 Skill 工具触发              | 输入 `/文件名`（去掉 .md 后缀）               |
| **适合场景** | 复杂的多步骤技能（如批改作业、生成报告）                  | 简单的一次性命令（如检查文档、转换格式）               |
| **复杂程度** | 较复杂，需要文件夹结构                           | 较简单，单文件即可                          |

> 🔑 **一句话记住**：
> - **Skills = 技能包**（文件夹结构，适合复杂流程）
> - **Commands = 斜杠命令**（单文件，适合简单命令）

你必须创建**文件夹**，文件夹里放一个 `SKILL.md` 文件：

```
C:\Users\你的用户名\.claude\skills\

      └── SKILL.md        ← 文件必须叫这个名字
```

**调用方式**：输入 `/grade` 或直接说"批改小作业"

你只要创建**单个文件**：

```
C:\Users\你的用户名\.claude\commands\
  └── check-dashboard.md   ← 直接创建文件，文件名就是命令名
```

**调用方式**：输入 `/check-dashboard`（去掉 .md 后缀）

## 写在前面：这份作业怎么用？


先完成实操演示的小练习4和小练习5，找不到的可以找助教老师


**场景拆解最常见的场景，提示词区分skill技能和command斜杠命令。**

**选你岗位的一项具体工作，同时做两个工具**：

- **Skills（技能包）**：把这项工作拆成标准步骤，以后按步骤走，不遗漏、不犯错
- **Commands（斜杠命令）**：对这项工作的成果做多维度的质量检查，确保产出达标

一个skill管"怎么做事"，一个command管"做得好不好"。**配套使用，工作流程和质量都有了保障。**

**把一个场景做透、做扎实**。

## 为什么要做这个作业？

### 这个作业解决什么真实问题？

| 你的困扰                       | 这份作业怎么解决                     |
| -------------------------- | ---------------------------- |
| 工作流程没有标准化，每次做法不一样，容易遗漏步骤   | Skills帮你把工作拆成标准步骤，每次照着走，不漏不乱 |
| 做完的东西质量不稳定，有时候好有时候差        | Commands帮你做系统性的质量检查，确保每次都达标  |
| 检查别人（或自己）的工作成果时，总是凭感觉，没有标准 | Commands是从多个维度量化检查，不是"感觉还行"  |
| 已有的 Skills 不够实用，用不起来       | 这次只聚焦一个场景，拆得更细，做完就能真的用       |

### 为什么是「做 + 查」配套，而不是两个独立的工具？

因为在真实工作中，**做事和检查是一体的**：

> 📖 **举个例子**：
>
> 财务岗做完月末报表，需要有人检查数据对不对——
> - **Skills（技能包）**：告诉你月末结账的标准流程是什么（第一步做什么、第二步做什么...）
> - **Commands（斜杠命令）**：告诉你报表做完后，从哪些维度检查（比如，格式的规划性、数据准确性、公式正确性、截止日期、合规性...）
>
> 如果只有 Skills，你知道怎么做，但做出来好不好不知道；
> 如果只有 Commands，你知道怎么查，但做的时候没有标准流程。
> **两个配套，才是一个完整的工作质量保障体系。**

### 做完你能收获什么？

- ✅ **1个真正能用的 Skills 技能包**——你岗位某项工作的标准执行步骤，以后照着走不遗漏
- ✅ **1个真正能用的 Commands 斜杠命令**——对这项工作成果的多维度质量检查，确保产出达标
- ✅ **掌握"做+查"配套设计的思维方式**——以后任何工作都可以用这个思路：先定步骤，再定检查标准
- ✅ **真正理解 Skills 和 Commands 的区别**——Skills 是"按步骤执行"，Commands 是"多维度评估"，两者协作大于单打独斗
- ✅ **巩固"拆场景→写提示词→测试优化"的核心能力**——这次只做一个场景，但做得比上个月更扎实
- ✅ **找到自己的"卡点"**——知道哪一步写不好提示词、哪一步测试结果不理想，比假装会更有价值

> ⚠️ **隐私提醒**：提交作业时，请将真实姓名、手机号、公司名称等敏感信息替换为化名或模糊描述（例如：用「某互联网公司」代替公司名，用「张XX」代替客户姓名，用模拟数据代替真实数据）。

## 开始之前：先选好你要做的工作场景

这是整个作业最关键的一步——**场景选得好，后面越做越顺；场景选得太大，后面做不下去。**

### 怎么选场景？

用这个标准来选：

| 标准 | 说明 |
|------|------|
| ✅ **高频** | 你每周或每月至少做1次的工作 |
| ✅ **有明确步骤** | 这件事不是纯创意性的，而是有流程可以拆的 |
| ✅ **有可检查的成果** | 做完之后有一个"产出物"可以检查质量 |
| ✅ **你足够熟悉** | 你做过很多次，知道每个环节的坑在哪 |

## Part 1：做 Skills（技能包）—— 把工作拆成标准步骤

> 📌 Skills 是技能包，需要创建文件夹结构。适合复杂的、多步骤的执行流程。

### Step 1：拆解工作流程（15分钟）

在开始写 Skills 之前，先用纸笔或文档工具，把你选的工作拆成步骤。

**📋 拆解模板（先自己填，不要用AI）：**

```
【工作名称】：___________________

步骤1：_______________
  - 关键动作：_______________
  - 容易出错的地方：_______________

步骤2：_______________
  - 关键动作：_______________
  - 容易出错的地方：_______________

步骤3：_______________
  - 关键动作：_______________
  - 容易出错的地方：_______________

步骤4：_______________
  - 关键动作：_______________
  - 容易出错的地方：_______________

步骤5：_______________
  - 关键动作：_______________
  - 容易出错的地方：_______________
```

> ⚠️ **重要：先自己拆，不要一开始就让AI帮你拆。** AI拆出来的步骤往往太笼统，不够贴合你的实际工作。你自己拆完之后，可以让AI帮你优化，但起点必须是你的真实经验。

## Part 2：做 Commands（斜杠命令）—— 多维度质量检查

> 📌 Commands 是斜杠命令，只需要创建单个文件。适合简单的检查命令。

### Step 5：设计检查维度（15分钟）

Commands 的核心是**从多个维度系统地检查工作成果的质量**，而不是凭感觉说"还行"。

**📋 检查维度设计模板：**

```
【工作成果名称】：___________________

我要从以下维度检查这项工作的质量：

维度1：【例如：数据准确性】
  - 检查什么：【关键数字是否正确，有无计算错误】
  - 怎么判断正确：【数据来源可追溯，计算公式无误】
  - 常见问题：【抄错数字、公式引用范围错误】

维度2：【例如：完整性】
  - 检查什么：【所有要求的要素是否都包含了】
  - 怎么判断正确：【对照清单逐项检查，无遗漏】
  - 常见问题：【漏掉某个子项目、缺签字、缺附件】

维度3：【例如：格式规范性】
  - 检查什么：【文档/报表的格式是否符合规范】
  - 怎么判断正确：【字体、字号、表格格式统一，页码齐全】
  - 常见问题：【格式不统一、缺页眉页脚、打印区域设置错误】

维度4：【例如：时效性】
  - 检查什么：【是否在规定时间内完成】
  - 怎么判断正确：【在截止日期前提交，且留有审核时间】
  - 常见问题：【卡在最后一刻提交，没有留出修改时间】

维度5：【例如：合规性/专业性】
  - 检查什么：【是否符合公司制度或行业规范】
  - 怎么判断正确：【符合相关制度和标准，无违规内容】
  - 常见问题：【超出权限范围、不符合最新政策】
```

> 💡 **至少设计5个维度，最多不超过8个。** 维度太多检查不完，维度太少检查不全面。

### Step 6：创建 Commands 文件（20分钟）

Commands 只需要创建单个文件。以下是具体步骤：

**📍 文件位置**：`C:\Users\你的用户名\.claude\commands\check-你的命令名.md`

**📋 操作步骤：**

1. 打开 Claude Code
2. 输入以下提示词，让 Claude 帮你创建 Commands 文件

**📋 提示词模板（直接复制到Claude Code）：**

```
请帮我创建一个 Commands 斜杠命令文件。这是一个质量检查命令。

【我的信息】
- 岗位：【例如：财务专员】
- 命令名称：【例如：check-month-end（检查月末报表）】
- 工作成果：【例如：月末财务报表】
- 检查人角色：【例如：财务主管/自查】

【检查维度】（请根据我的输入设计详细的检查逻辑）

维度1：【数据准确性】
  检查要点：【关键数字是否正确、计算公式是否无误、数据来源是否可追溯】
  评分标准：【全部正确=通过，发现1处错误=需修正，发现3处以上=需重做】

维度2：【完整性】
  检查要点：【所有要求的报表/表格是否齐全、签字是否完整、附件是否齐全】
  评分标准：【全部齐全=通过，缺少1项=需补充，缺少3项以上=需重做】

维度3：【格式规范性】
  检查要点：【格式统一、标题清晰、表格对齐、页码齐全】
  评分标准：【全部规范=通过，存在格式问题=需调整】

维度4：【时效性】
  检查要点：【是否在截止日前完成、是否留有审核时间】
  评分标准：【按时完成=通过，逾期=需说明原因】

维度5：【合规性】
  检查要点：【是否符合公司财务制度、是否符合会计准则】
```

## 避坑清单

| 问题 | 原因 | 解决方法 |
|------|------|---------|
| 场景选得太大，拆不出步骤 | 选了一个"大而全"的场景 | 缩小范围！例如把"做月报"缩小到"月末结账这一个环节" |
| Skills 的步骤太笼统 | 只写了"做什么"，没写"怎么做"和"注意什么" | 每个步骤补充具体的操作细节和常见错误 |
| Commands 的维度太空泛 | 检查标准不具体，无法量化 | 每个维度给出"通过/需修正/需重做"的具体判断标准 |
| 测试时 AI 输出不理想 | 提示词不够具体 | 在提示词中加入你的岗位信息和具体工作背景 |
| Skills 和 Commands 不配套 | 两个工具的场景不一致 | 确保它们针对的是**同一个工作**的不同阶段 |
| Commands 检查不出问题 | 维度设计不全面 | 补充维度，或把现有维度拆得更细 |
| 不知道文件放哪里 | 混淆了 Skills 和 Commands 的目录 | Skills 放 `skills/文件夹名/`，Commands 放 `commands/` |
| 文件命名不对 | 用了中文或特殊字符 | 用英文或拼音命名（如 `month-end`、`check-month-end`） |

## Skills 和 Commands 的区别（关键知识回顾）

很多同学分不清 Skills 和 Commands 的区别，这里再强调一次：

| 维度 | Skills（技能包） | Commands（斜杠命令） |
|------|-----------------|-------------------|
| **是什么** | 一套完整的技能包 | 一个简单的斜杠命令 |
| **目录位置** | `~/.claude/skills/文件夹名/` | `~/.claude/commands/` |
| **文件结构** | 文件夹 + 里面的 `SKILL.md` 文件 | 单个 `.md` 文件 |
| **命名方式** | 文件夹名（如 `month-end`） | 文件名（如 `check-month-end.md`） |
| **调用方式** | `/文件夹名` 或触发关键词 | `/文件名`（去掉.md） |
| **用途** | 告诉你"怎么做"（执行步骤） | 告诉你"做得好不好"（质量检查） |
| **触发时机** | 做事的时候 | 做完之后检查 |
| **类比** | 菜谱（做菜的步骤） | 美食评委的评分标准 |

> 💡 **一句话记忆**：Skills 是"师傅教你做"，Commands 是"质检员帮你查"。
