## 8.1 缺陷分类决策树

当 SIT 中字段验证失败时，使用此决策树确定缺陷类型和责任方：

```
验证失败
│
├─ 期望值在 Origination 中存在，但在 Odessa 中错误
│  └─ 集成缺陷
│     责任方: 乙方
│     修复: Origination→Odessa 接口中的数据映射逻辑或 API 转换
│
├─ 期望值在 Origination 中错误（但在 QT 中正确）
│  └─ Origination 缺陷
│     责任方: 乙方
│     修复: QT→Origination 映射或 Origination 业务逻辑
│
├─ 期望值在 QT 本身中错误
│  └─ 测试数据问题
│     责任方: 甲方
│     修复: 更新 Integration Data 表中的正确测试数据
│
├─ 默认值不正确（例如：中登 ≠ Pending，Serviced ≠ Yes）
│  └─ 配置缺陷
│     责任方: 乙方
│     修复: 系统配置或种子数据设置
│     参考: 文档 02 第 2.2 节完整默认值列表
│
└─ 计算值不正确（例如：到期日偏差 1 天）
   └─ 计算缺陷
      责任方: 乙方
      修复: 应用代码中的公式逻辑
      参考: 文档 02 第 2.3 节预期公式
```

### 缺陷优先级指南

| 优先级 | 定义 | 示例 |
|---|---|---|
| P1 - 严重 | 阻塞所有测试执行或导致数据损坏 | 合同完全无法落地到 Odessa；还款计划完全缺失 |
| P2 - 高 | 特定合同类型或字段类别异常 | 所有浮动利率字段显示为固定值；Fundings 标签页完全为空 |
| P3 - 中 | 单个字段不正确但存在变通方案 | Tax Code 显示错误值；Manufacturer 解析不正确 |
| P4 - 低 | 外观或非功能性问题 | 显示格式问题；字段标签不匹配 |

## 8.2 当前测试执行状态

### Deal 状态汇总（来自 Deals Automated 表）

| 状态 | 数量 | 占比 | 所需操作 |
|---|---|---|---|
| ✅ 已完成 + 已落地到 Odessa | 7 | 18% | 作为基准/参考测试用例使用 |
| ⚠️ 已完成 + 落地失败 | 6 | 16% | 集成缺陷 — 优先进行根因分析 |
| ⚠️ 工作流未完成 | 9 | 24% | 在业务工作流中被阻塞 — 检查信用审批或检查清单是否未完成 |
| 🔴 尚无状态 | 16 | 42% | 尚未执行 — 优先待办 |
| **合计** | **38** | **100%** | |

### 关键发现

1. **42% 的测试用例尚未开始** — 这是主要的进度风险
2. **6 个 Deal 已落地但验证失败** — 表明存在系统性集成缺陷，需要在运行更多测试用例之前进行根因分析
3. **9 个 Deal 卡在工作流中** — 可能被信用审批（B13）或检查清单完成所阻塞；需与业务团队确认
4. **仅 7 个完全成功的 Deal** — 不足以完成 SIT 签收；每种合同类型至少需要 1 个成功的 Deal

### 建议优先级

1. **立即处理**: 对 6 个落地失败的 Deal 进行根因分析 — 可能存在影响多个测试用例的共性缺陷模式
2. **本周完成**: 解除 9 个工作流未完成 Deal 的阻塞 — 协调业务方完成审批
3. **下个 Sprint**: 执行 16 个未触及的测试用例，理想情况下使用文档 07 中的 Tier 1 自动化方案

来自全球模板的 92 个回归场景。当前分类：

| 状态 | 数量 | 详情 |

| 部门 | 适用场景 | 数量 |

## 8.4 知识库维护协议

| 触发条件 | 操作 | 责任人 | 方式 |
|---|---|---|---|
| 发现新缺陷 | 使用分类（8.1）和根因添加到缺陷日志 | QA Lead | 缺陷跟踪工具 |
| 乙方提出知识库中没有的问题 | 回答 → 记录 → 添加到相应知识库文档 | 甲方 PM | 按需更新文档 01-05 |
| 系统字段新增或重命名 | 更新文档 04 或 05 中的字段映射 | 甲方 PM + BA | 将发布说明与当前知识库对比 |
| 业务规则变更 | 更新文档 02，获取相关方签字确认，通知乙方 | 甲方 PM | 文档 02 变更日志 |
| 新增合同类型 | 添加到文档 01 合同类型表，更新 Integration Data | 甲方 PM | 文档 01 + 源 Excel |
| Sprint 回顾发现知识空白 | 创建新章节或扩展现有文档 | 甲方 PM | 知识库中的任意文档 |
| 测试用例已自动化 | 更新文档 06 操作表中的分类 | 甲方 PM | 文档 06 |

### 季度健康检查

每季度（或在重大里程碑时）检查：
- [ ] 文档 04-05 中的所有字段映射是否仍然准确？
- [ ] 文档 02 中的业务规则是否有变更但未更新知识库？
- [ ] 文档 07 中的自动化分类是否仍然有效？
- [ ] 是否有反复出现的乙方问题应添加到知识库中？
- [ ] 缺陷分类决策树（8.1）是否仍然完整？
