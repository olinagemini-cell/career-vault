## 9.1 复用指南 — 适配未来项目

本知识库从设计上即支持复用。10份文档的结构、操作分类方法论以及项目管理框架具有通用性，每个项目仅需更改各文档中的具体内容。

### 需要替换的内容（国家/项目特定内容）

| 文档 | 变更内容 | 示例 |
|---|---|---|
| 01 - 产品与合同类型 | 产品类型、合同类型、活动值 | "Finance Lease China" → "Finance Lease Vietnam" |
| 02 - 业务规则 | 税率、币种、法律实体、监管规则、默认值、公式 | 增值税 13% → GST 10%；CNY → VND；中登 → 当地登记机构 |
| 03 - 角色与流程 | 第三方角色定义（如有差异）、审批工作流阶段 | 可能有不同的审批链或额外参与方 |
| 04 - QT → Origination 映射 | 字段名称、可编辑/锁定状态（如系统版本不同） | 可能新增字段，也可能移除部分字段 |
| 05 - Origination → Odessa 映射 | 字段名称、校验规则、默认值、条件逻辑 | 所有校验规则均与国家相关 |
| 06 - 操作手册 | 操作步骤（如 UI 不同）、输入数据来源 | UI 导航可能因系统版本不同而有差异 |
| 08 - 项目管理 | 测试执行状态、缺陷数量、回归场景清单 | 每个项目均为全新数据 |

### 保持不变的内容（通用框架）

| 文档 | 保留内容 | 原因 |
|---|---|---|
| 00 - 索引与导航 | 10份文档结构、阅读路径、交叉引用指南 | 框架与具体系统无关 |
| 07 - 自动化与 AI 策略 | 分类方法论（AI/混合/人工）、分层结构、ROI 模型 | 该方法适用于任何多系统集成 |
| 08 - 项目管理 | 缺陷分类树结构、维护协议、周报模板 | 通用项目管理实践 |
| 09 - 复用模板 | 本指南、术语表结构、版本历史格式 | 元框架 |

### 新项目适配检查清单

- [ ] **步骤 1**：将全部 10 份文档复制到新项目文件夹
- [ ] **步骤 2**：在文档 00 中更新新项目名称、系统信息、团队联系方式
- [ ] **步骤 3**：将文档 01 表格中的所有值替换为新国家/产品数据
- [ ] **步骤 4**：根据新的业务需求和监管规则重建文档 02
- [ ] **步骤 5**：如参与方角色或流程阶段有差异，更新文档 03
- [ ] **步骤 6**：根据新系统字段规格文档重建文档 04-05
- [ ] **步骤 7**：对新的端到端测试用例重新执行操作分解 → 更新文档 06
- [ ] **步骤 8**：根据新系统的技术栈，重新评估文档 07 中的自动化工具选型
- [ ] **步骤 9**：为新项目重置文档 08 的状态跟踪
- [ ] **步骤 10**：在文档 09 术语表中补充新术语

**预估适配工作量**：对于范围相似的项目（相同系统平台、新国家），需 3-5 天。如系统平台也发生变更，则需 1-2 周。

## 9.2 术语表

### 业务术语

| 术语 | 全称 | 定义 |
|---|---|---|
| DL | Direct Lease（直接租赁） | 出租人从供应商处购买资产并租赁给客户 |
| SLB | Sale-and-Leaseback（售后回租） | 客户将现有资产出售给出租人并回租使用 |
| AOR | Asset Operating Rental（资产经营性租赁） | 经营性租赁 / 无担保贷款结构 |
| HP | Hire Purchase（分期付款购买） | 客户在支付最后一期款项后获得资产所有权（香港） |
| ORV | Optional Residual Value（可选残值） | 租赁到期时客户有权以残值购买资产的租赁方式 |
| PG | Personal Guarantor（个人担保人） | 为租赁提供个人担保的自然人 |
| CG | Corporate Guarantor（企业担保人） | 为租赁提供企业担保的法人 |
| COF | Cost of Funds（资金成本） | 代表出租人融资成本的基准利率 |
| LPR | Loan Prime Rate（贷款市场报价利率） | 中国的基准浮动利率指数，由中国人民银行发布 |
| IRR | Internal Rate of Return（内部收益率） | 在 Quotation Tool 中计算的盈利能力指标 |
| IM | International Manufacturer（国际制造商） | 采购渠道：通过国际制造商的本地实体采购资产 |

### 系统术语

| 术语 | 全称 | 定义 |
|---|---|---|
| QT | Quotation Tool（报价工具） | 用于创建租赁报价和模拟的前端系统 |
| Origination | Origination System（业务发起系统） | 用于交易结构化、审批和合规的中台系统 |
| Odessa | Odessa Back Office（Odessa 后台） | 用于合同管理、账单和会计的后台系统 |
| SGEF | Société Générale Equipment Finance | 母公司平台 / 集成端点 |
| TP Ref | Third Party Reference（第三方引用） | 系统中参与方（客户、供应商、经销商）的唯一标识符 |
| AMT | Asset Management Tool（资产管理工具） | 提供型号代码的资产参考系统 |

### 技术术语

| 术语 | 全称 | 定义 |
|---|---|---|
| SIT | System Integration Testing（系统集成测试） | 验证多个关联系统间数据流的测试阶段 |
| E2E | End-to-End（端到端） | 从第一个系统（QT）到最后一个系统（Odessa）的完整流程 |
| RPA | Robotic Process Automation（机器人流程自动化） | 用于重复性 UI 操作的软件自动化技术 |
| UAT | User Acceptance Testing（用户验收测试） | 由业务用户验证系统行为的测试阶段 |
| KB | Knowledge Base（知识库） | 供团队参考的结构化项目知识仓库 |

### 中国监管术语

| 术语 | 定义 |
|---|---|
| 中登 (ZhongDeng) | 中国动产登记系统。融资租赁合同须在此登记以获得法律保护 |
| 增值税 (VAT) | 增值税。中国有形动产租赁标准税率为 13% |
| 统一社会信用代码 | 中国企业法人的 18 位统一标识码（登记证件类型 00715） |

## 9.5 反馈与改进

改进本知识库的建议：

1. **乙方团队**：当你遇到知识库未能解答的问题时，不要仅仅向 PM 提问，还应记录你期望在哪份文档中找到答案。这有助于我们将新内容放置到正确的位置。
2. **甲方 PM**：每个迭代结束后，审查缺陷日志中是否存在指向知识库内容缺失的模式。反复出现的问题 = 缺失的知识库条目。
3. **未来项目**：完成适配（第 9.1 节）后，记录哪些部分比预期更困难，并为下一个团队更新适配检查清单。
