## 进阶路径：从场景到工作流

**重要提醒：不要一开始就想做大，先做小，逐步沉淀。**

### 第一步：先做小（单个场景）

```
目标：跑通1个场景
├── 从5个场景中选1个最简单的
├── 写一个提示词，测试效果
├── 反复用几次，验证有效
└── 产出：1个可用的提示词
```

### 第二步：逐个训练沉淀

```
目标：每个场景都沉淀成MD文档
├── 每个场景反复使用
├── 记录哪里好用、哪里不好用
├── 优化提示词（加背景、加格式要求、加示例）
├── 沉淀成一份完整的MD文档
└── 产出：5份场景MD文档
```

### 第三步：汇聚成工作流

```
目标：多个场景串联成完整流程
├── 把相关的场景串起来
├── 设计场景之间的衔接
├── 形成完整的业务工作流
└── 产出：1个业务工作流（可以是Skills/Agent）
```

### 示例：销售经理的进阶路径

```
第一步：先做"客户背景调研"这1个场景
        ↓
第二步：逐个沉淀5个场景的MD文档
        - 客户背景调研.md
        - 客户分级评估.md
        - 拜访准备清单.md
        - 需求分析报告.md
        - 销售周报生成.md
        ↓
第三步：串联成"销售全流程AI助手"
        线索 → 调研 → 分级 → 拜访准备 → 需求分析 → 周报
```

### 关键心态

- **不要贪多**：先把1个场景做到好用，再做下一个
- **持续迭代**：每个场景都需要用几次才能优化好
- **记录问题**：每次用完记录哪里不满意，下次改进
- **先跑通再优化**：60分的方案先用起来，比100分的方案躺在那里强

## 常见问题

### Q1：我说不清楚我的岗位职责怎么办？
A：让AI帮你。告诉它你的岗位名称、行业、公司类型，让它参考招聘网站的JD帮你梳理。然后你对照着确认哪些是你实际在做的。

### Q2：我的工作很杂，没有明确的模块怎么办？
A：先记录一周你都做了什么，然后让AI帮你归类。通常都能归到5-8个模块里。

### Q3：我找不到5个场景怎么办？
A：每个工作模块至少能找到1个场景。如果某个模块找不到，可能是这个模块本身就不适合AI赋能（比如纯体力活、纯人际关系）。

### Q4：我的场景太简单/太复杂怎么办？
A：简单的场景先做，容易出成果。复杂的场景可以拆成多个简单场景。

### Q5：怎么判断一个场景值不值得做？
A：问自己：这件事我每周花多少时间？如果AI能帮我省一半时间，一年能省多少小时？如果答案超过50小时，就值得做。
