# 战略分析型汇报（精华版）

---

## 适用场景
- 业务规划/年度战略汇报
- 新业务机会评估
- 战略复盘与调整

---

## 核心框架：战略五问

```
1. Where to play? → 在哪个市场/领域竞争
2. How to win? → 如何赢得竞争
3. What capabilities? → 需要什么核心能力
4. What management systems? → 需要什么管理体系
5. What will we measure? → 如何衡量成功
```

---

## 汇报结构（10-15页PPT）

### Part 1: 市场洞察（2-3页）
```
1. 市场规模：TAM/SAM/SOM
2. 增长趋势：CAGR + 驱动因素
3. 竞争格局：主要玩家 + 市场份额
4. 机会窗口：为什么是现在
```

### Part 2: 战略选择（3-4页）
```
1. 核心矛盾：我们面临的关键抉择
2. 选项分析：2-3个战略选项对比
3. 推荐方案：选择X，因为...
4. 风险应对：主要风险 + 缓释措施
```

### Part 3: 执行路径（3-4页）
```
1. 关键举措：3-5个战略行动
2. 资源需求：人/财/时间
3. 里程碑：阶段目标 + 时间节点
4. 成功指标：KPI + 目标值
```

### Part 4: 请求决策（1页）
```
今天需要决策的事项：
1. [具体决策1]
2. [具体决策2]
```

---

## 分析工具速查

| 工具 | 用途 | 核心问题 |
|------|------|---------|
| 3C分析 | 机会评估 | 客户要什么/竞争对手做什么/我们能做什么 |
| SWOT | 战略定位 | 优势/劣势/机会/威胁 |
| 波特五力 | 行业分析 | 竞争强度/进入壁垒/替代威胁 |
| 价值链 | 能力分析 | 哪个环节创造最大价值 |

---

## 常见陷阱与对策

| 陷阱 | 对策 |
|------|------|
| 数据堆砌无结论 | 每页必有"So What" |
| 战略太空泛 | 具体到"做什么/不做什么" |
| 忽略执行难度 | 明确资源需求和风险 |
| 没有决策请求 | 最后一页必须有请求 |
