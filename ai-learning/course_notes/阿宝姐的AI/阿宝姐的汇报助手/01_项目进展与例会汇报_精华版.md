# 项目进展与例会汇报（精华版）

---

## 核心框架：STAR-P

| 要素 | 说明 | 时间占比 |
|------|------|---------|
| **S**ituation | 背景/上期回顾 | 10% |
| **T**ask | 本期目标 | 10% |
| **A**ction | 关键动作/进展 | 40% |
| **R**esult | 成果/数据 | 30% |
| **P**lan | 下一步计划 | 10% |

---

## 例会汇报三版本

### 30秒版（电梯汇报）
```
本周完成了[X]，达成率[Y]%，下周重点推进[Z]。
```

### 3分钟版（站会）
```
1. 上周目标：[目标]
2. 完成情况：[结果] + [关键数据]
3. 遇到问题：[问题] → [解决方案/需要支持]
4. 本周计划：[3个重点]
```

### 10分钟版（周会）
```
1. 回顾（2min）：上期目标 vs 实际完成
2. 亮点（3min）：关键突破 + 可复制经验
3. 问题（2min）：卡点 + 解决思路
4. 计划（2min）：本期目标 + 里程碑
5. 支持（1min）：需要的资源/协助
```

---

## 进度汇报公式

### 正常进度
```
【项目名】进度[X]%，符合预期，预计[日期]完成
```

### 延期风险
```
【项目名】当前[X]%，存在[Y]风险，建议：
1. 方案A：[调整范围] → 按期交付
2. 方案B：[延期X天] → 保证质量
推荐方案[X]，原因是[Z]
```

### 超额完成
```
【项目名】提前完成，节省[X]资源
关键因素：[可复制经验]
建议：[下一步行动]
```

## 常见问题处理

| 场景 | 话术模板 |
|------|---------|
| 被追问细节 | "这个问题的关键是...，我可以会后发详细数据" |
| 进度落后 | "确实有延迟，根因是X，已采取Y措施，预计Z恢复" |
| 资源不足 | "按当前资源预计X结果，如增加Y可达到Z" |
| 方向质疑 | "您的顾虑很有道理，我们考虑过...，选择当前方案是因为..." |
