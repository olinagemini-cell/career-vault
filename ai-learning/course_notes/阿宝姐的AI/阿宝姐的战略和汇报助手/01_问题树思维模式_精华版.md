# 问题树思维模式（精华版）

---

## 核心概念
**问题树（Issue Tree）**：将复杂问题拆解为可管理的子问题
- 从一个核心问题出发，逐层分解
- 找到问题根因或机会点
- 聚焦关键路径

---

## MECE原则

**MECE** = Mutually Exclusive, Collectively Exhaustive
- **相互独立**：各部分之间没有重叠
- **完全穷尽**：所有部分加起来=完整整体

```
【符合MECE】
      总体
    ┌──┴──┐
    A      B
  ┌─┼─┐  ┌─┼─┐
  A1 A2  B1 B2
✅ A和B相互独立、A+B完全穷尽
```

### MECE自检
- [ ] 各分支是否相互独立？
- [ ] 所有分支是否等于整体？
- [ ] 可用公式验证？（收入-成本=利润）
- [ ] 分支数量3-7个？

---

## 问题树构建步骤

```
第1步：明确核心问题（精准、可量化）
  ↓
第2步：第一层分解（2-5个主要分支，MECE）
  ↓
第3步：逐层细化（每层3-5个子分支）
  ↓
第4步：验证完整性（是否遗漏？是否重叠？）
  ↓
第5步：聚焦关键（找到最重要的2-3个分支）
```

---

## 经典拆解模式

### 模式1：是否式（Yes/No Tree）
```
核心问题
├─ 是（成立）→ 怎么办？
└─ 否（不成立）→ 怎么办？
```

### 模式2：要素式（Component Tree）
```
核心问题
├─ 维度A（如：收入端）
├─ 维度B（如：成本端）
└─ 维度C（如：效率端）
```

### 模式3：流程式（Process Tree）
```
核心问题
├─ 阶段1（获客）
├─ 阶段2（转化）
├─ 阶段3（交付）
└─ 阶段4（复购）
```

## 实战案例：提升盈利能力

```
❌ 错误拆解：
├─ 增加销售额
├─ 提高产品价格 ← 与上面重叠
├─ 降低成本
└─ 提升客户满意度 ← 不直接贡献盈利

✅ 正确拆解（MECE）：
盈利能力提升
├─ 增加收入（开源）
│   ├─ 增加销量
│   │   ├─ 刺激需求
│   │   └─ 提升份额
│   └─ 提高客单价
│       ├─ 优化定价
│       └─ 改善产品组合
└─ 降低成本（节流）
    ├─ 降低供应成本
    └─ 降低运营成本
```

## 分析优先级

| 优先级 | 判断标准 |
|--------|----------|
| 高 | 影响大+可控+数据支撑 |
| 中 | 影响中等或数据不足 |
| 低 | 影响小或不可控 |

---

## 核心要点
1. **先发散再聚焦**：第一层尽量MECE，然后找关键分支深入
2. **数据驱动**：用数据判断哪个分支最重要
3. **假设先行**：有初步假设再验证，不是漫无目的拆解
4. **80/20法则**：20%的因素贡献80%的结果

```markdown

【我的问题】：[输入问题]
【背景信息】：[输入背景]

【你需要做的】：
1. 用MECE原则构建问题树（3层）
2. 每层给出分解逻辑
3. 标注最值得深入的分支
4. 输出Markdown格式的问题树
```
