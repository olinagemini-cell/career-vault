# 决策矩阵分析法（精华版）

---

## 核心概念

**决策矩阵**：用2x2矩阵快速定位优先级

```
              业务价值
           低         高
可    高 ┌─────┬─────┐
行        │ 考虑 │ 优先 │
性        │      │ 执行 │
      低 ├─────┼─────┤
         │ 暂缓 │ 深入 │
         │      │ 评估 │
         └─────┴─────┘
```

---

## 两大核心维度

### 维度1：业务价值（Why值得做）
```
业务价值评估
├─ 收入影响（能带来多少营收？）
├─ 成本节约（能省多少钱？）
├─ 用户体验（提升多少满意度？）
├─ 战略意义（对长期发展影响？）
└─ 紧迫程度（不做会怎样？）
```

### 维度2：可行性（How能做成）
```
可行性评估
├─ 技术难度（技术能实现吗？）
├─ 资源需求（需要多少人/钱/时间？）
├─ 风险程度（有哪些不确定性？）
├─ 依赖关系（需要哪些前置条件？）
└─ 团队能力（团队能搞定吗？）
```

---

## 四象限决策

| 象限 | 特征 | 决策 |
|------|------|------|
| **优先执行** | 高价值+高可行 | 立即行动 |
| **深入评估** | 高价值+低可行 | 探索降低难度 |
| **考虑做** | 低价值+高可行 | 资源空闲时做 |
| **暂缓** | 低价值+低可行 | 不做或推迟 |

---

## 评分方法

### 简单评分（1-5分）
| 维度 | 1分 | 3分 | 5分 |
|------|-----|-----|-----|
| 业务价值 | 可有可无 | 有一定价值 | 战略必须 |
| 可行性 | 几乎不可能 | 有挑战 | 很容易 |

### 决策规则
- 业务价值≥4 且 可行性≥4 → 优先执行
- 业务价值≥4 且 可行性<4 → 深入评估
- 业务价值<4 且 可行性≥4 → 考虑做
- 业务价值<4 且 可行性<4 → 暂缓

## 实战案例：产品功能优先级

```
【待评估功能】
A. AI智能推荐
B. 社群互动功能
C. 数据看板
D. 会员等级体系

【评分】
| 功能 | 业务价值 | 可行性 | 象限 |
|------|---------|--------|------|
| A.AI推荐 | 5分 | 3分 | 深入评估 |
| B.社群互动 | 3分 | 4分 | 考虑做 |
| C.数据看板 | 4分 | 5分 | 优先执行 |
| D.会员等级 | 4分 | 4分 | 优先执行 |

【决策】
1. 先做C和D（高价值+高可行）
2. 深入评估A（探索技术方案降低难度）
3. B排到后面（资源空闲时做）
```

---

## 进阶：加权评分

当需要更精确时，可加权：

```
业务价值 = 收入影响×40% + 战略意义×30% + 紧迫度×30%
可行性 = 技术难度×40% + 资源需求×30% + 风险×30%
```

---

## 常见陷阱

| 陷阱 | 说明 | 解决 |
|------|------|------|
| 高估可行性 | 低估技术难度 | 让技术团队评估 |
| 低估价值 | 只看短期收益 | 考虑战略意义 |
| 主观偏见 | 凭感觉打分 | 用数据支撑 |
| 忽略依赖 | 遗漏前置条件 | 画依赖关系图 |

---

## 与其他框架配合

```
问题树 → 拆解问题、列出选项
    ↓
决策矩阵 → 评估优先级
    ↓
战略解码阶梯 → 落地执行
```

## AI实战提示词

```markdown
请你扮演：[产品顾问]，用决策矩阵帮我排优先级。

【待评估选项】：
1. [选项A]
2. [选项B]
3. [选项C]

【背景信息】：[输入背景]

【你需要做的】：
1. 定义业务价值和可行性的评估维度
2. 对每个选项打分（1-5分）
3. 画出2x2矩阵，标注各选项位置
4. 给出优先级排序和执行建议
```

---
**核心理念**：资源有限，必须选择；选择的本质是取舍
