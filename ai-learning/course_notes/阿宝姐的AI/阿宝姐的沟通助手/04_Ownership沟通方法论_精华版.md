# Ownership沟通方法论（精华版）

---

## 核心理念
**Ownership = 对结果负责**，不是只看层级关系，而是看"我要什么结果"

---

## 方法论1: 沟通目的导向

### 正确思考顺序
```
第1步: 明确目的 → "我要对方做什么？"
可能目的：只是告知 / 需要决策 / 需要协调 / 需要指导

第2步: 站对方角度 → "对方需要什么信息？"
必须提供：背景 / 现状 / 我的判断 / 需要什么决策

第3步: 准备信息再沟通 → "我要给什么信息才能让对方推进？"
准备清单：背景(简洁) + 现状(客观) + 判断(有依据) + 方案(if有) + 需求(明确)
```

### 常见错误
❌ 没思考就抛问题："领导，同事A联系了我的作者，怎么办？"
✅ 带着思考去找领导："领导，我遇到这样一个情况...[背景+现状+判断+想法+需要确认]"

## 方法论2: Ownership意识

### 两种思维对比
| 只有层级概念(❌) | 有Ownership意识(✅) |
|-----------------|-------------------|
| "A是中层，我能直接找她吗？" | "我需要这个信息，A有，我能不能搞定？" |
| 只考虑层级关系 | 先考虑目的，再考虑手段，最后考虑层级 |
| 被动等安排 | 主动获取信息 |

### 6步思考法
```
1. 我要获得什么信息？
2. 这个信息能帮我做什么决定？
3. 这个信息谁有？
4. 我去找这个人，对方会不会有顾虑？
5. 我能不能设计沟通方式打消顾虑？
6. 如果处理不了，需要向上借力吗？
```

### 判断标准：什么时候自己去 vs 向上借力
| 维度 | 自己去 | 向上借力 |
|------|--------|---------|
| 敏感度 | 低(只了解情况) | 高(涉及利益冲突) |
| 层级差 | 小(平级/高一级) | 大(高两级以上) |
| 把握度 | 有把握打消顾虑 | 没把握/可能说错话 |

## 方法论3: 向上借力Storyline

### 3分钟汇报框架
```
【背景】30秒 - 我在做什么事，发生了什么情况
【现状】30秒 - 现在是什么情况，我了解到什么
【我的判断】30秒 - 可能有哪些情况，依据是什么
【我的方案】1分钟 - 我打算怎么做，为什么这样做
【需要您】30秒 - 需要您做什么决策/给几个选项
```

### 完整话术示例
```
"领导，我遇到这样一个情况：

【背景】我在跟XX项目，作者说同事A联系了她

【现状】A说是社里的意思，主要问数据库的事，但我不确定是否影响我的项目

【我的判断】可能两种情况：
1. 高层安排，A只是了解情况
2. 涉及项目调整，需要协调

【我的想法】我打算先和A了解情况，看具体目的，再决定下一步

【需要您确认】您这边是否已了解？还是我先去了解再汇报？"
```

## 友好获取信息话术

### 只是了解情况(不涉及冲突)
```
"XX，您好！
我听说您最近联系了XX，想了解一下情况。
因为我这边也在跟这个项目，想确认咱们是不是有需要协调的？
方便的话能简单聊几句吗？"
```

### 可能涉及冲突(试探性)
```
"XX，我这边在推进XX项目，听说您那边也在接触这个方向？
我想了解：咱们是不是同一个项目？还是不同业务线？有没有需要我配合的？
避免咱们重复劳动或有冲突~"
```

## 核心金句
> "不是你不能借力领导，但你得带着自己的思考去让领导参与，而不是没思考就把问题抛给领导。"

> "不是只看层级，而是看'我要什么信息''我能不能搞定'。能搞定就自己去，搞不定就向上借力。"

---

## 常见错误对比
| 错误 | 正确 |
|------|------|
| "领导，同事A联系了我的作者，怎么办？" | "[背景+现状+判断+方案]您看我这样处理可以吗？" |
| "A是中层，我不能直接找她吧？" | "我需要这个信息，A有，我能不能友好地去问？" |
| "领导，我想和A了解情况" | "我想和A了解[什么情况]，因为[为什么需要]，我打算[怎么问]，您看可以吗？" |
