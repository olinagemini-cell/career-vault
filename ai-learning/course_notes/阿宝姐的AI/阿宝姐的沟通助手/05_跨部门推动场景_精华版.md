# 跨部门推动场景（精华版）

---

## 场景定义
**编码**: B-2-01 | **难度**: ⭐⭐⭐
**核心矛盾**: 需协调无直接汇报关系的部门配合，对方缺乏动力或优先级不高

---

## 核心框架：四层推进法
```
【第一层：升维思考】站在业务角度理解价值
     ↓
【第二层：系统分析】多方案对比，逻辑完整
     ↓
【第三层：灵活推动】向上借力 + 横向协作 + 动态决策
     ↓
【第四层：结构化沟通】对齐目标→呈现方案→给建议→保持开放
```

---

## 方法论1: 升维思考

### ❌ 常见错误
- 只站在自己部门角度看困难
- 陷入技术细节："维护成本高"
- 没理解业务价值

### ✅ 正确做法
```
必须回答：
1. 直接价值是什么？（对业务/产品）
2. 间接价值是什么？（对公司长期发展）
3. 对老板/部门/公司/协作方各有什么价值？

核心原则：困难可解决，关键是值不值得做
```

---

## 方法论2: 系统分析

### 多方案对比框架
```
【目标】要实现的业务价值

【方案A】
├─ 实现路径 / 优势 / 劣势
├─ 投入(时间/人力) / 风险
└─ 适用场景

【方案B】[同上结构]

【方案C】[同上结构]

【建议】综合对比后倾向XX，理由是...但可讨论
```

## 方法论3: 向上借力

### 情形1: 资源不足(30%→100%策略)
```
第一步: 和直属领导align方向（至少不反对）
话术: "我想先用现有资源做个pilot，验证可行性，您看可以吗？"

第二步: 用30%资源做MVP
→ 做出showcase，用结果说话

第三步: 用showcase争取更大资源
→ 利用公开平台自然展示，让高层"自然看到"
```

### 情形2: 搞定求稳型领导(螺旋上升)
```
阶段1: 先在舒适区给成绩
"领导，我先帮您把XX目标搞定"

阶段2: 小步突破(可控风险)
"基于成功经验，我想尝试XX，风险可控，您看可以吗？"

阶段3: 持续给成绩，螺旋上升
```

### 情形3: 避免领导边缘化焦虑
```
四维度平衡：
- 能力OK（能干活）
- 帮领导完成诉求（有价值）
- 对外低调（威胁小）
- 功劳上交（领导有安全感）
```

### 当老板不支持时
```
1. 分析为什么不支持（方案没击中诉求/跨部门压力/信息不同）
2. 完善方案再沟通（强调对老板的好处/降低风险）
3. 如仍不支持，接受但留记录（"那咱们这么做，但可能有XX风险"）
4. 持续收集信息，等待时机
```

## 方法论4: A↔B动态推进

### 核心理解
不是A→B线性，而是A↔B交替迭代

### 判断依据
| 参数 | 自己先推动 | 先找老板 |
|------|-----------|---------|
| 风险 | 低(常规流程) | 高(涉及资源调动) |
| 信息 | 不足(需收集) | 充足(有完整方案) |

## 方法论5: 转化语言(和Tech沟通)

### 五步法
```
1. 放低门槛让对方先说
"我对这块不太懂，能请教一下吗？"

2. 脑中带着假设(预判对方在意的风险)

3. 把对方的话抽象成逻辑框架
"我理解你主要顾虑是：1.技术复杂度 2.维护成本 3.系统稳定性，对吗？"

4. 推演业务影响
"那这些技术风险会影响哪些业务点？"

5. 起到桥梁作用
"我理解tech担心XX，能不能这样[折中方案]？"
```

---

## 方法论6: Story telling

### 核心逻辑
```
1. 站在对方角度出发 → 理解需求/价值/痛点
2. 营造需求(起点) → "我理解您的XXX"
3. 呈现方案(过程) → 用讲故事的方式
4. 达成目标(结果) → 对方愿意接受
```

## 话术模板库

### 模板1: 降低门槛+建立合作
```
"XX，知道你们最近特别忙。
我这边有个XX想讨论一下可行性。
[如果对方说忙] 没问题，先聊聊思路。"
```

### 模板2: 讲清价值+建立共识
```
"这个项目主要为了[业务目标]。
因为咱们[大老板/公司]要做[战略方向]。
对咱们能[业务价值]；对你们也是[对方价值]。"
```

### 模板3: 感受困难+引导表达
```
"咱们的难度主要在哪里？
是[顾虑A]？还是[顾虑B]？或者[顾虑C]？"
```

### 模板4: 探测边界
```
"如果我[调整A]，你能节约什么？
如果我[调整B]，你最快能达到什么程度？"
```

### 模板5: 30%资源做MVP
```
"领导，我理解现在资源有限。
能不能先给我30%资源做个pilot，验证可行性？
成功了再决定是否投入更多。风险可控，您看可以吗？"
```

## 核心原则
> "一个人愿意支持你，一定是有利益绑定的。
> 人永远为自己收获的信息和决策买单。"
