## 场景1: 向老板申请资源
**方法论组合**: PREP + 关键信息三件套 + 换位思考

### 话术模板
```
【P-结论】"老板，我建议Q2增加50万预算，用于新渠道拓展。"

【R-理由】"主要基于三点：
1. Q1测试ROI达1:4.2
2. 竞品正加大投放
3. Q2是销售旺季"

【E-展开(三件套)】
[数据] "Q1投120万获506万，ROI超行业平均"
[案例] "竞品A本月预算增80%"
[证言] "CMO联盟张总说Q2是最佳窗口"

【P-重申】"50万投入预计带来210万增量，建议批准。"
```

### 避坑
- ❌ 只说"我需要资源"
- ✅ 要说"投资回报"
- ❌ 只讲困难
- ✅ 要讲机会和风险

## 场景2: 工作汇报
**方法论组合**: PREP + B-F-H话语权掌控

### 话术模板
```
【开场-结论先行】
"老板，本月目标达成率105%，主要三个突破：
1) 新渠道转化提升40%
2) 老客复购率翻倍
3) 成本下降20%
您想先听哪部分？"

【被打断时-B-F-H】
老板："数据怎么来的？"
你："这是关键问题(Bridge)，数据来源是CRM(快速回答)，
不过我特别想强调(Flag)，这背后反映的趋势(Hook)：
客户质量明显提升，不只是数量增长..."
```

### 关键
- 准备3个版本：简版3分钟/标准10分钟/详版20分钟
- 被打断不慌张，用B-F-H拉回主线

## 场景3: 跨部门推动
**方法论组合**: 换位思考 + 拜托前5问 + 言语柔顺剂

### 准备阶段
```
【换位思考】技术部的KPI是什么？难处在哪？

【拜托前5问】
1. 动机：能帮他证明技术价值
2. 能力：技术可行，不算复杂
3. 麻烦：需要2天开发
4. 时机：避开版本发布前
5. 铺垫：先请教技术可行性
```

### 沟通话术
```
【开场-柔顺剂】
"张工，知道你们版本发布很忙(共情)，
我有个技术问题想先请教(铺垫)..."

【转入需求】
"如果能实现，可帮公司提升20%转化率(对公司价值)。"

【降低麻烦】
"我已整理好详细文档(减少工作量)，
可以排Q2后半段(给时间)，
上线后会在全员会特别感谢(公开认可)。"

【给选择题】
"你看是Q2后半段，还是Q3初比较合适？"
```

## 场景4: 给下属负面反馈
**方法论**: 有温度的反馈6步法

### 话术模板
```
【1-陈述事实】
"小王，这次汇报PPT有几页数据错误，客户也质疑了。"

【2-说明影响】
"导致：1)客户质疑专业度 2)团队要重新核对"

【3-询问原因】
"是时间太紧，还是对数据来源不清楚？"

【4-讨论改进】
"建议：1)汇报前提前1天给我过 2)数据让数据组复核"

【5-表达信任】
"我相信你的能力，下次一定能做得更好。"

【6-感谢对话】
"谢谢你愿意坦诚沟通，有需要随时找我。"
```

### 避坑
- ❌ 说"你总是"、"你从来"
- ❌ 公开场合批评
- ✅ 对事不对人
- ✅ 共创解决方案

## 场景5: 请同事帮忙
**方法论**: 拜托前5问 + 言语柔顺剂

### 话术模板
```
【肯定对方】
"李哥，上次你做的XX项目思路特别清晰，团队都在学习。"

【解释背景】
"我们部门遇到类似问题，跟你之前做的很像。"

【清晰请求】
"想请你帮忙看看方案，给些建议。"

【承诺回报】
"我已整理好材料，大概占用半小时，
下次你们需要市场数据我第一时间支持，
项目成功后周会上特别感谢。"
```

---

## 使用原则
1. **先诊断场景** - 是向上/跨部门/冲突/反馈？
2. **确定核心方法** - 选1-2个最适合的
3. **添加辅助方法** - 根据情况补充
4. **灵活调整** - 理解逻辑，不死板套用
